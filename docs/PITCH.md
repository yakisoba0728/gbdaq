# GBDAQ — 1-Pager

**예측시장을 5분 만에 띄우는 엔진을 팝니다.** GBDAQ(지비닥)는 LMSR 가격 책정 엔진 + 실제 LLM 애널리스트 + 폴리마켓풍 라이브 UX를 하나로 묶은 화이트라벨 예측시장입니다. 지금 보이는 교내 데모는 그 엔진의 쇼케이스일 뿐입니다.

## 문제 / 기회

조직에는 "여론을 한눈에 보고 싶은 질문"이 늘 있습니다 — 학교는 학생 참여, 회사는 출시일·마감일 전망. 설문은 한 번 찍고 끝이라 시간에 따라 변하는 확률을 못 보여주고, 직접 예측시장을 만들려면 가격 엔진·분석·실시간 UI를 처음부터 짜야 합니다. GBDAQ는 그 세 가지를 이미 만들어 두었고, 주제만 바꾸면 어디든 올라갑니다.

## 타깃 — 누가 왜 돈을 내나

| 타깃 | 무엇을 산다 | 왜 돈을 내나 |
| --- | --- | --- |
| **(a) 학교·동아리·행사 (B2B/B2G)** | 화이트라벨 예측시장 인스턴스 | 축제·체육대회·선거 같은 행사에 참여형 소통툴을 빠르게 띄우고 싶다. 직접 개발할 여력은 없고, 플레이머니라 안전하다. |
| **(b) 사내 의사결정 예측시장** | 멀티테넌트 호스팅 시트 | 프로젝트 마감·출시일·KPI 달성 여부를 사내 인원의 베팅으로 *forecasting*한다. 기업 내부 예측시장(internal prediction market)은 실재하는 카테고리로, 설문보다 시간에 따른 확률 변화를 잘 잡아낸다. |
| **(c) 엔진/SDK 라이선스** | LMSR + LLM 애널리스트 엔진 | 자체 제품에 마켓메이킹·확률 가격·LLM 분석을 붙이려는 팀이, 검증된 엔진을 사서 개발 시간을 아낀다. |

## 차별점

오픈소스 베팅 위젯은 많지만, **LMSR 정밀 가격 + 실제 LLM 애널리스트 + 라이브 UX**를 한 번에 주는 화이트라벨은 드뭅니다. GBDAQ의 LMSR 엔진(`lib/lmsr.ts`)은 매수 비용·금액당 지분(closed-form 역산)·매도 수령액을 순수 함수로 계산하고 단위 테스트로 검증되며, AI 애널리스트는 실제 Claude(`claude-haiku-4-5`)를 서버 전용 라우트(`app/api/analyze/route.ts`)에서 forced tool use로 호출합니다.

## 로드맵

1. **라이브 데모(현재)** — 교내 마켓, 클라이언트 가격 엔진, `localStorage` 단일 기기, 실제 Claude 애널리스트(+ 휴리스틱 폴백).
2. **멀티테넌트 / 주최자 대시보드** — 워크스페이스·마켓 생성/관리, 사용자 공유 상태(서버 영속).
3. **정산 · 분석 리포트** — 마켓 resolution(정산), 참여·정확도 집계, 주최자용 리포트.
4. **엔진 SDK 공개** — LMSR + LLM 애널리스트 파이프라인을 임베드 가능한 패키지로.

## 규제 · 안전

전 구간 **플레이머니**(상점)이며 **현금으로 환전되지 않습니다.** 베팅 대가로 돈이 오가지 않으므로 사행성(도박) 규제에 해당하지 않고, 미성년 사용자에게도 안전합니다. 매출은 베팅하는 참여자가 아니라 **B2B/B2G 주최자·라이선시**에게서만 발생합니다.
